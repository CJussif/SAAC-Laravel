con un sistema para
actividades complementarias

04
Objetivo
Propuesta de solución
Desarrollar una
plataforma web
integral para el TESCHa
  • Gestión centralizada de actividades, cupos y horarios por el administrador.
  • Control de asistencia y acreditación en tiempo real por docentes.
  • Acceso de alumnos a su historial, constancias en PDF y subida de evidencias.
  • Flujo de invitación por correo — registro solo para usuarios autorizados.

07
Admin
Módulo administrador
Control total
del sistema
📚 Catálogo de actividades
Crear, editar y eliminar actividades. Asignar docente, cupo, horario y créditos.
👥 Gestión de alumnos
Ver perfil, carrera, semestre y ajustar créditos acumulados manualmente.
✅ Validar evidencias
Revisar solicitudes pendientes. Aprobar o rechazar con motivo. Notificación automática al alumno.
📄 Panel de constancias
Historial de constancias emitidas. Filtro por nombre, matrícula o tipo. Exportar CSV.
✉️ Invitaciones
Registro cerrado. El admin envía invitaciones por correo con token único de activación.
📊 Dashboard KPIs
Resumen de actividades activas, alumnos inscritos y constancias emitidas del ciclo.
08
Docente
Módulo docente
Seguimiento
de asistencia
en tiempo real
  • Pase de lista — Selecciona fecha, marca presentes y ausentes por alumno.
  • Mis grupos — Vista de todas las actividades asignadas con conteo de inscritos, barra de asistencia por alumno y enlace a expediente.
  • Acreditar alumno — Una vez que el alumno cumple asistencia mínima, el docente lo acredita y el sistema genera la constancia automáticamente.
  • Expedientes — Tabla completa con % asistencia, estatus, sesiones. Alerta de alumnos en riesgo (<60%). Exportar CSV.
09
Alumno
Módulo alumno
Todo el historial
académico
en un solo lugar
  • Catálogo e inscripción — Consulta actividades disponibles, cupo y horario. Inscribirse con un clic (validación de cupo y duplicados).
  • Panel principal — Muestra actividades en curso, créditos acumulados y notificaciones no leídas.
  • Subir evidencia — Registrar actividades externas (eventos, cursos, competencias) con archivo comprobante. El admin valida y otorga créditos.
  • Mis constancias — Lista de actividades acreditadas. Botón de descarga genera PDF al instante con folio único, firma y datos del alumno.

13
Conclusiones
Lo que aprendimos
Un proyecto
que resuelve
un problema real
  • Inertia.js elimina la complejidad de un API REST separado manteniendo las ventajas de React como SPA.
  • Tailwind v4 con tokens personalizados (guinda, cream) garantiza coherencia visual sin CSS manual.
  • TrustProxies en Laravel es indispensable para desplegar detrás de Cloudflare — las URLs se generan correctamente con HTTPS.
  • Las pruebas automatizadas (82 tests) dieron confianza para refactorizar sin romper funcionalidades existentes.
